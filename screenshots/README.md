# Screenshots for Business Intelligence Dashboard

This folder contains screenshots of the application for documentation and portfolio purposes.

## Folder Structure

```
screenshots/
├── desktop/          # Desktop browser screenshots (1920x1080 or similar)
└── mobile/           # Mobile device screenshots (375x667 or similar)
```

## Naming Convention

Please use descriptive names for your screenshots:

### Desktop Screenshots:
- `dashboard-main.png` - Main dashboard overview
- `dashboard-charts.png` - Dashboard with charts visible
- `sales-management.png` - Sales page with data table
- `sales-filtered.png` - Sales page with filters applied
- `inventory-management.png` - Inventory page overview
- `inventory-add-product.png` - Add product modal/form
- `inventory-edit-product.png` - Edit product modal
- `login-interface.png` - Login page
- `reports-page.png` - Reports/analytics page

### Mobile Screenshots:
- `dashboard-mobile.png` - Mobile dashboard view
- `mobile-navigation.png` - Mobile menu/navigation
- `sales-mobile.png` - Sales page on mobile
- `inventory-mobile.png` - Inventory page on mobile
- `login-mobile.png` - Login page on mobile

## Screenshot Guidelines

### Desktop:
- Browser: Chrome/Firefox latest version
- Resolution: 1920x1080 or higher
- Show full browser window with address bar
- Capture key functionality in action

### Mobile:
- Use browser dev tools mobile simulation
- iPhone SE (375x667) or similar small screen
- Show touch-friendly interface
- Capture swipe gestures if applicable

## Adding Screenshots

1. Take screenshots of your running application
2. Save them in the appropriate folder with descriptive names
3. Commit and push to update the README with actual images

The README.md will automatically reference these images once they're added!
